The max-value entropy search acquisition function

Open in GitHubRun in Google Colab

The max value entropy search acquisition function​

Max-value entropy search (MES) acquisition function quantifies the information gain about the maximum of a black-box function by observing this black-box function fff at the candidate set x{\textbf{x}}x (see [1, 2]). BoTorch provides implementations of the MES acquisition function and its multi-fidelity (MF) version with support for trace observations. In this tutorial, we explain at a high level how the MES acquisition function works, its implementation in BoTorch and how to use the MES acquisition function to query the next point in the optimization process.

In general, we recommend using Ax for a simple BO setup like this one, since this will simplify your setup (including the amount of code you need to write) considerably. You can use a custom BoTorch model and acquisition function in Ax, following the Using BoTorch with Ax tutorial. To use the MES acquisition function, it is sufficient to add "botorch_acqf_class": qMaxValueEntropy, to model_kwargs. The linked tutorial shows how to use a custom BoTorch model. If you'd like to let Ax choose which model to use based on the properties of the search space, you can skip the surrogate argument in model_kwargs.

1. MES acquisition function for q=1q=1q=1 with noisy observation​

For illustrative purposes, we focus in this section on the non-q-batch-mode case (q=1q=1q=1). We also assume that the evaluation of the black-box function is noisy. Let us first introduce some notation:

  • f∗=max⁡X(f(x))f^* = \max_\mathcal{X} (f(\textbf{x}))f∗=maxX​(f(x)), the maximum of the black-box function f(x)f(\textbf{x})f(x) in the design space X\mathcal{X}X
  • y=f(x)+ϵ,ϵ∼N(0,σϵ2)y = f(\textbf{x}) + \epsilon, \epsilon \sim N(0, \sigma^2_\epsilon)y=f(x)+ϵ,ϵ∼N(0,σϵ2​), the noisy observation at the design point x\textbf{x}x
  • h(Y)=EY[−log⁡(p(y))]=−∫Yp(y)log⁡p(y)dyh(Y) = \mathbb{E}_Y[-\log(p(y))] = -\int_\mathcal{Y} p(y)\log p(y) dyh(Y)=EY​[−log(p(y))]=−∫Y​p(y)logp(y)dy, the differential entropy of random variable YYY with support Y\mathcal{Y}Y: the larger is h(Y)h(Y)h(Y), the larger is the uncertainty of YYY.
  • v(D)=−ED[h(F∗∣D)]v(\mathcal{D}) = -\mathbb{E}_D[h(F^*\mid\mathcal{D})]v(D)=−ED​[h(F∗∣D)], the value of data set D\mathcal{D}D, where F∗F^*F∗ denotes the function maximum (a random variable in our context of our model).

The Max-value Entropy Search (MES) acquisition function at x\textbf{x}x after observing Dt\mathcal{D}_tDt​ can be written as

αMES(x)=v(Dt∪(x,y))−v(Dt)=−EY[h(F∗∣Dt∪(x,Y))]+h(F∗∣Dt)=−EY[h(F∗∣Y)]+h(F∗)=I(F∗;Y)=I(Y;F∗)(symmetry)=−EF∗[h(Y∣F∗)]+h(Y)\begin{align} \alpha_{\text{MES}}(\textbf{x}) &= v(\mathcal{D}_t\cup {(\textbf{x}, y)}) - v(\mathcal{D}_t) \\ &= - \mathbb{E}_Y[h(F^* \mid \mathcal{D}_t\cup {(\textbf{x}, Y)})] + h(F^*\mid\mathcal{D}_t) \\ &= - \mathbb{E}_Y[h(F^* \mid Y)] + h(F^*) \\ &= I(F^*; Y) \\ &= I(Y; F^*) \quad \text{(symmetry)} \\ &= - \mathbb{E}_{F^*}[h(Y \mid F^*)] + h(Y) \\ \end{align}αMES​(x)​=v(Dt​∪(x,y))−v(Dt​)=−EY​[h(F∗∣Dt​∪(x,Y))]+h(F∗∣Dt​)=−EY​[h(F∗∣Y)]+h(F∗)=I(F∗;Y)=I(Y;F∗)(symmetry)=−EF∗​[h(Y∣F∗)]+h(Y)​​

, which is the mutual information of random variables F∗∣DtF^*\mid \mathcal{D}_tF∗∣Dt​ and Y∣x,DtY \mid \textbf{x}, \mathcal{D}_tY∣x,Dt​. Here F∗F^*F∗ follows the max value distribution conditioned on Dt\mathcal{D}_tDt​, and YYY follows the GP posterior distribution with noise at x\textbf{x}x after observing Dt\mathcal{D}_tDt​.

Rewrite the above formula as

αMES(x)=−H1+H0,H0=h(Y)=log⁡(2πe(σf2+σϵ2))H1=EF∗[h(Y∣F∗)]≃1∣F∗∣ΣF∗h(Y∣f∗))\begin{align} \alpha_{\text{MES}}(\textbf{x}) &= - H_1 + H_0, \\ H_0 &= h(Y) = \log \left(\sqrt{2\pi e (\sigma_f^2 + \sigma_\epsilon^2)}\right) \\ H_1 &= \mathbb{E}_{F^*}[h(Y \mid F^*)] \\ &\simeq \frac{1}{\left|\mathcal{F}_*\right|} \Sigma_{\mathcal{F}_*} h(Y\mid f^*)) \end{align}αMES​(x)H0​H1​​=−H1​+H0​,=h(Y)=log(2πe(σf2​+σϵ2​)​)=EF∗​[h(Y∣F∗)]≃∣F∗​∣1​ΣF∗​​h(Y∣f∗))​​

, where F∗\mathcal{F}_*F∗​ are the max value samples drawn from the posterior after observing Dt\mathcal{D}_tDt​. Without noise, p(y∣f∗)=p(f∣f≤f∗)p(y \mid f^*) = p(f \mid f \leq f^*)p(y∣f∗)=p(f∣f≤f∗) is a truncated normal distribution with an analytic expression for its entropy. With noise, Y∣F≤f∗Y\mid F\leq f^*Y∣F≤f∗ is not a truncated normal distribution anymore. The question is then how to compute h(Y∣f∗)h(Y\mid f^*)h(Y∣f∗) or equivalently p(y∣f≤f∗)p(y\mid f \leq f^*)p(y∣f≤f∗)?

Using Bayes' theorem,

p(y∣f≤f∗)=P(f≤f∗∣y)p(y)P(f≤f∗)\begin{align} p(y\mid f \leq f^*) = \frac{P(f \leq f^* \mid y) p(y)}{P(f \leq f^* )} \end{align}p(y∣f≤f∗)=P(f≤f∗)P(f≤f∗∣y)p(y)​​​

, where

  • p(y)p(y)p(y) is the posterior probability density function (PDF) with observation noise.
  • P(f≤f∗)P(f \leq f^*)P(f≤f∗) is the posterior cummulative distribution function (CDF) without observation noise, given any f∗f^*f∗.

We also know from the GP predictive distribution

[yf]∼N([μμ],[σf2+σϵ2σf2σf2σf2]).\begin{align} \begin{bmatrix} y \\ f \end{bmatrix} \sim \mathcal{N} \left( \begin{bmatrix} \mu \\ \mu \end{bmatrix} , \begin{bmatrix} \sigma_f^2 + \sigma_\epsilon^2 & \sigma_f^2 \\ \sigma_f^2 & \sigma_f^2 \end{bmatrix} \right). \end{align}[yf​]∼N([μμ​],[σf2​+σϵ2​σf2​​σf2​σf2​​]).​​

So

f∣y∼N(u,s2)\begin{align} f \mid y \sim \mathcal{N} (u, s^2) \end{align}f∣y∼N(u,s2)​​

, where

u=σf2(y−μ)σf2+σϵ2+μs2=σf2−(σf2)2σf2+σϵ2=σf2σϵ2σf2+σϵ2\begin{align} u &= \frac{\sigma_f^2(y-\mu)}{\sigma_f^2 + \sigma_\epsilon^2} + \mu \\ s^2 &= \sigma_f^2 - \frac{(\sigma_f^2)^2}{\sigma_f^2 + \sigma_\epsilon^2} = \frac{\sigma_f^2\sigma_\epsilon^2}{\sigma_f^2 + \sigma_\epsilon^2} \end{align}us2​=σf2​+σϵ2​σf2​(y−μ)​+μ=σf2​−σf2​+σϵ2​(σf2​)2​=σf2​+σϵ2​σf2​σϵ2​​​​

Thus, P(f≤f∗∣y)P(f \leq f^* \mid y)P(f≤f∗∣y) is the CDF of above Gaussian.

Finally, given f∗f^*f∗, we have

h(Y∣f∗)=−∫Yp(y∣f∗)log⁡(p(y∣f∗))dy=−∫YZp(y)log⁡(Zp(y))dy≃−1∣Y∣ΣYZlog⁡(Zp(y)),Z=P(f≤f∗∣y)P(f≤f∗)\begin{align} h(Y \mid f^*) &= -\int_\mathcal{Y} p(y \mid f^*)\log(p(y \mid f^*)) dy\\ &= -\int_\mathcal{Y} Zp(y)\log(Zp(y)) dy \\ &\simeq -\frac{1}{\left|\mathcal{Y}\right|} \Sigma_{\mathcal{Y}} Z\log(Zp(y)), \\ Z &= \frac{P(f \leq f^* \mid y)}{P(f \leq f^* )} \end{align}h(Y∣f∗)Z​=−∫Y​p(y∣f∗)log(p(y∣f∗))dy=−∫Y​Zp(y)log(Zp(y))dy≃−∣Y∣1​ΣY​Zlog(Zp(y)),=P(f≤f∗)P(f≤f∗∣y)​​​

, where ZZZ is the ratio of two CDFs and Y\mathcal{Y}Y is the samples drawn from the posterior distribution with noisy observation. The above formulation for noisy MES is inspired from the MF-MES formulation proposed by Takeno et. al [1], which is essentially the same as what is outlined above.

Putting all together,

αMES(x)=H0−H1≃H0−H1MC=log⁡(2πe(σf2+σϵ2))+1∣F∗∣ΣF∗1∣Y∣ΣY(Zlog⁡Z+Zlog⁡p(y))\begin{align} \alpha_{\text{MES}}(\textbf{x}) &= H_0 - H_1 \\ &\simeq H_0 - H_1^{MC}\\ &= \log \left(\sqrt{2\pi e (\sigma_f^2 + \sigma_\epsilon^2)}\right) + \frac{1}{\left|\mathcal{F}^*\right|} \Sigma_{\mathcal{F}^*} \frac{1}{\left|\mathcal{Y}\right|} \Sigma_{\mathcal{Y}} (Z\log Z + Z\log p(y)) \end{align}αMES​(x)​=H0​−H1​≃H0​−H1MC​=log(2πe(σf2​+σϵ2​)​)+∣F∗∣1​ΣF∗​∣Y∣1​ΣY​(ZlogZ+Zlogp(y))​​

The next design point to query is chosen as the point that maximizes this aquisition function, i. e.,

xnext=max⁡x∈XαMES(x)\begin{align} \textbf{x}_{\text{next}} = \max_{\textbf{x} \in \mathcal{X}} \alpha_{\text{MES}}(\textbf{x}) \end{align}xnext​=x∈Xmax​αMES​(x)​​

The implementation in Botorch basically follows the above formulation for both non-MF and MF cases. One difference is that, in order to reduce the variance of the MC estimator for H1H_1H1​, we apply also regression adjustment to get an estimation of H1H_1H1​,

H^1=H1MC−β(H0MC−H0)\begin{align} \widehat{H}_1 &= H_1^{MC} - \beta (H_0^{MC} - H_0) \end{align}H1​​=H1MC​−β(H0MC​−H0​)​​

, where

H0MC=−1∣Y∣ΣYlog⁡p(y)β=Cov(h1,h0)Var(h1)Var(h0)h0=−log⁡p(y)h1=−Zlog⁡(Zp(y))\begin{align} H_0^{MC} &= - \frac{1}{\left|\mathcal{Y}\right|} \Sigma_{\mathcal{Y}} \log p(y) \\ \beta &= \frac{Cov(h_1, h_0)}{\sqrt{Var(h_1)Var(h_0)}} \\ h_0 &= -\log p(y) \\ h_1 &= -Z\log(Zp(y)) \\ \end{align}H0MC​βh0​h1​​=−∣Y∣1​ΣY​logp(y)=Var(h1​)Var(h0​)​Cov(h1​,h0​)​=−logp(y)=−Zlog(Zp(y))​​

This turns out to reduce the variance of the acquisition value by a significant factor, especially when the acquisition value is small, hence making the algorithm numerically more stable.

For the case of q>1q > 1q>1, joint optimization becomes difficult, since the q-batch-mode MES acquisiton function becomes not tractable due to the multivariate normal CDF functions in ZZZ. Instead, the MES acquisition optimization is solved sequentially and using fantasies, i. e., we generate one point each time and when we try to generate the iii-th point, we condition the models on the i−1i-1i−1 points generated prior to this (using the i−1i-1i−1 points as fantasies).

2. Setting up a toy model​

We will fit a standard SingleTaskGP model on noisy observations of the synthetic 2D Branin function on the hypercube [−5,10]×[0,15][-5,10]\times [0, 15][−5,10]×[0,15].

import math
import torch

from botorch.test_functions import Branin
from botorch.fit import fit_gpytorch_mll
from botorch.models import SingleTaskGP
from botorch.utils.transforms import standardize, normalize
from gpytorch.mlls import ExactMarginalLogLikelihood

torch.manual_seed(7)

bounds = torch.tensor(Branin._bounds).T
train_X = bounds[0] + (bounds[1] - bounds[0]) * torch.rand(10, 2)
train_Y = Branin(negate=True)(train_X).unsqueeze(-1)

train_X = normalize(train_X, bounds=bounds)
train_Y = standardize(train_Y + 0.05 * torch.randn_like(train_Y))

model = SingleTaskGP(train_X, train_Y)
mll = ExactMarginalLogLikelihood(model.likelihood, model)
fit_gpytorch_mll(mll);

3. Defining the MES acquisition function​

The qMaxValueEntropy acquisition function is a subclass of MCAcquisitionFunction and supports pending points X_pending. Required arguments for the constructor are model and candidate_set (the discretized candidate points in the design space that will be used to draw max value samples). There are also other optional parameters, such as number of max value samples F∗\mathcal{F^*}F∗, number of Y\mathcal{Y}Y samples and number of fantasies (in case of q>1q>1q>1). Two different sampling algorithms are supported for the max value samples: the discretized Thompson sampling and the Gumbel sampling introduced in [2]. Gumbel sampling is the default choice in the acquisition function.

from botorch.acquisition.max_value_entropy_search import qMaxValueEntropy

candidate_set = torch.rand(
    1000, bounds.size(1), device=bounds.device, dtype=bounds.dtype
)
candidate_set = bounds[0] + (bounds[1] - bounds[0]) * candidate_set
qMES = qMaxValueEntropy(model, candidate_set)

4. Optimizing the MES acquisition function to get the next candidate points​

In order to obtain the next candidate point(s) to query, we need to optimize the acquisition function over the design space. For q=1q=1q=1 case, we can simply call the optimize_acqf function in the library. At q>1q>1q>1, due to the intractability of the aquisition function in this case, we need to use either sequential or cyclic optimization (multiple cycles of sequential optimization).

from botorch.optim import optimize_acqf

# for q = 1
candidates, acq_value = optimize_acqf(
    acq_function=qMES,
    bounds=bounds,
    q=1,
    num_restarts=10,
    raw_samples=512,
)
candidates, acq_value
Out: 
(tensor([[1.5350, 0.0758]]), tensor(0.0121))
# for q = 2, sequential optimization
candidates_q2, acq_value_q2 = optimize_acqf(
    acq_function=qMES,
    bounds=bounds,
    q=2,
    num_restarts=10,
    raw_samples=512,
    sequential=True,
)
candidates_q2, acq_value_q2
Out: 
(tensor([[-0.3238,  0.6565],
       [ 1.5349,  0.0748]]), tensor([0.0135, 0.0065]))
from botorch.optim import optimize_acqf_cyclic

# for q = 2, cyclic optimization
candidates_q2_cyclic, acq_value_q2_cyclic = optimize_acqf_cyclic(
    acq_function=qMES,
    bounds=bounds,
    q=2,
    num_restarts=10,
    raw_samples=512,
    cyclic_options={"maxiter": 2},
)
candidates_q2_cyclic, acq_value_q2_cyclic
Out: 
(tensor([[-0.3236,  0.6563],
       [ 1.5326,  0.0732]]), tensor([0.0101, 0.0064]))

The use of the qMultiFidelityMaxValueEntropy acquisition function is very similar to qMaxValueEntropy, but requires additional optional arguments related to the fidelity and cost models. We will provide more details on the MF-MES acquisition function in a separate tutorial.
